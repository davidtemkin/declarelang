# Tenets

The **tenets** are what Declare commits to being and doing — the promises the
positioning makes and the code must keep. They are the *intent* half of the
platform's two truths: the **code** is what Declare *does*; the **tenets** are
what it has *promised* to do. When the two disagree, the code is what drifted —
a tenet is not softened to match a diminished reality; it is either restored in
code or, deliberately, retired.

## Where they come from

The tenets are **interpretively distilled** from the language's own positioning —
its most load-bearing written commitments:

- the homepage's on-page copy,
- the "Why a new language, now?" essay,
- the FAQ,
- `docs/declare.md` — the whole language, in one file.

Distilled, not extracted: a human reads those and decides what is actually
*promised*. (The sample apps making little use of image assets, for instance, is
**not** a promise of "no assets" — nobody wrote that, so it is not a tenet.) Once
distilled they are **ratified** and stand on their own; when the positioning
shifts, they are re-distilled and re-ratified — never auto-regenerated. They are
delicate: they are the brand's word, changed only deliberately.

They are not the **reference** (what exists and how it behaves — derived from
code), and not `system-design/` (why decisions were reached — background,
non-binding).

## Who reads them

The primary consumer is the prose-consistency audit, which runs *top-down*: it
holds the platform up against these promises and reports where the code has
stopped keeping one. Humans read them the same way — as the standard the language
is measured against.

## The five

1. [What Declare is](1%20ROTAS.md) — its nature: a language, a standing structure, renderer-independent.
2. [How you write it](2%20OPERA.md) — the two delimiters, TypeScript, no CSS, composition.
3. [The standing model](3%20TENET.md) — reactivity and continuity: the core idea.
4. [The wager](4%20AREPO.md) — small, verifiable, built for the era of machine-written code.
5. [How it ships](5%20SATOR.md) — compile-anywhere, static extraction, no build step, and an honest posture.
