# Docs alignment — audit & map (handoff for the IA/editorial pass)

**Status:** read-only inventory, 2026-07-14 (Opus prep for the Fable IA/editorial pass).
Companion to [`documentation.md`](documentation.md) (the governance contract). Nothing here
is executed — it's the scoped map so the alignment pass (governance §8 bootstrap) opens with
inventory in hand and spends its budget on judgment.

> ⚠ **The tree is moving under this audit.** A distro reorg is live: `dist-browser`→`bundles`,
> `web`→`browser`, `/examples` serving retired (the program URL is the sole run address),
> `examples/site`→`examples/homepage`, `examples/docs/model.json`→`docs-model.json`. Treat
> every path below as provisional; the layout work targets the **settled** post-reorg tree.

| artifact | where | size | state |

**The core problem restated:** spec + brief + guide are three parallel copies of the truth,
none generated from another, all free to drift. The reference is the one facet already
generated (and 68%-covered). No unified core doc exists yet.

## 2. Reference coverage (the coverage-gate backlog)

`docs-model.json`: **212 nodes** — 113 attributes, 57 methods, 23 events, 19 classes.
**145 have prose (68%); 67 do not.** That 67 is the concrete `@api`-prose writing task
(governance §3 mode 1): terse, per-element, code-adjacent → Opus/collab work, not the Fable
grind. Voice/template for it is a Fable call; the fill is systematic.

## 3. Drift findings (concrete, from a scan of `docs/`)

- **17 guide/doc files link into `design/`** — every one breaks when `design/`→`docs/system-design/`. Must be re-pointed (ideally to `declare-docs:` symbolic links, not paths, so this never recurs).
- **2 files carry pre-rename `Declare`/`Declare` residue** — stale since the Declare rename.

None catastrophic individually; together they confirm nothing keeps the prose true today.

## 4. Guide coverage map — covered vs MISSING

**Covered (18 chapters):** overview, tutorial, composition, constraints, prevailing, events,
states, layout, data, scope-nouns, formatting, animation, text+markdown, sizing, fonts,
input+focus, shipping, docs-internals.

**Missing / thin — the write-new list:**
- **Standard library / components** — *the grievance, confirmed*: Button/Checkbox/Slider/… are **mentioned in passing in 5 chapters, taught in none.** No first-class components chapter. High priority (it's a headline feature).
- **Verify — "how to check your program"** — the whole verify ladder (a user-facing concept) has no chapter.
- **Capabilities** — the enumerable env contract (`docs/system-design/capabilities.md`) — no user doc.
- **SEO / static extraction** — shipped feature, no guide coverage.
- **Getting started / install / run** — thin; scattered between overview and shipping; deserves a clean front door (operational).
- **Two-way binding / editors** — partial under `26-data`; may want its own treatment.

## 6. Target `docs/` shape (proposed; layout is the Fable pass's call)

```
docs/
  <core-doc>.md            spec + rationale + LLM brief (the "whole surface" artifact)
  guide/                   narrative learning path (derivative, declare-docs: linked)
  reference/               generated per-element pages (from @api)
  operational/             getting started, dev server, production build, CLIs, flags
  system-design/           CATEGORY A (this audit, the contract, the merged design/ record)
```

Ordering for the rewrite (governance §8): **core-doc-first → reference (fill @api) → guide**.
